## Core Principles

- **Rigor always** — the [`/rigor`] standard applies to all analytical work without being asked. Never fabricate parameters, functional forms, or numbers; cite sources; flag assumptions. Auto-loaded by the `rigor-inject` hook.
- **No fabrication** ([HARD RULE](rules/no-fabrication.md)) — every factual/numeric/code/citation claim in prose is verified against the source *in the current turn* before being written. No paraphrasing from memory.
- **Literature trail** ([HARD RULE](rules/lit-review-protocol.md)) — every paper read updates `lit_review.md`; consult it before re-reading or citing.
- **Plan first** — enter plan mode for non-trivial work (a new section, a revision round, a re-analysis); save plans to `.claude/quality_reports/plans/`.
- **Verify after** — compile (3-pass + bibtex), confirm the PDF, and resolve every note-to-self before declaring done. See [verification-protocol](rules/verification-protocol.md).
- **Replicate before extending** (empirical) — [replication-protocol](rules/replication-protocol.md): match published results to tolerance before building on them.
- **Quality gates** — nothing ships below 80/100 (90 for submission-ready).
- **`[LEARN]` tags** — when corrected, append `[LEARN:category] wrong → right (why)` to [MEMORY.md](MEMORY.md).

## Skills Quick Reference

This workflow ships with the two rigor skills from this repo's `skills/` library:

| Command | What it does |
|---|---|
| `/rigor` | The research-rigor standard — never fabricate parameters, cite sources, verify against primary texts (auto-loaded each session by the `rigor-inject` hook) |
| `/empirical-coding-discipline` | Audit-every-step empirical coding — verify units and identifiers, never brute-force |

The real machinery of this workflow is its **hard rules** (below). For a general manuscript toolkit — literature search, multi-pass paper review, bibliography validation, R analysis — see **Other helpful resources** in the repo README; those tools install alongside this config and complement it.

To compile: 3-pass `pdflatex`/`xelatex` + `bibtex` (see [verification-protocol](rules/verification-protocol.md)).

## What carries this workflow: the hard rules

1. **[no-fabrication.md](rules/no-fabrication.md)** — every factual claim verified against its source in the current turn, with a mechanical self-check (pattern triggers, post-draft audit, incident log).
2. **[lit-review-protocol.md](rules/lit-review-protocol.md)** — every paper read creates a `lit_review.md` entry *before* it may be cited.
3. **[replication-protocol.md](rules/replication-protocol.md)** — reproduce published numbers to tolerance before extending (empirical).
4. **[paper-writing-conventions.md](rules/paper-writing-conventions.md)** — pin the notation registry on day one (theory papers especially).
