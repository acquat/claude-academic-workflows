# Administrative Workflow — template

Claude as an **administrative assistant**, on two tracks: **(a) reimbursements** — receipt intake, claim forms, compliance, deadlines, the ledger; and **(b) document maintenance** — keeping versioned documents (CV, bio, teaching statement) up to date by pulling from your own records, never fabricating. Distilled from a working reimbursement-processing setup plus a cross-folder CV-maintenance workflow. **Not** a research/analysis config.

## What's inside

```
.claude/
├── CLAUDE.md                     # the brain — role, core loop, defaults, quality gates, non-negotiables
├── rules/
│   ├── policy.md                 # YOUR institution's rulebook — generated by /ingest-policy (or hand-filled)
│   ├── quality-gates.md          # form-completeness scoring rubric
│   ├── verification-protocol.md  # end-of-claim checklist (incl. hardcoded-totals check)
│   ├── document-maintenance.md   # (documents) pull-from-sources / never-fabricate / confirm / version
│   ├── plan-first-workflow.md · orchestrator-protocol.md · session-logging.md   # shared core
├── agents/
│   ├── policy-parser.md          # read an institution's policy doc → structured rulebook (read-only)
│   ├── receipt-classifier.md     # extract vendor/date/amount/category from a receipt
│   ├── form-filler.md            # fill the xlsx via openpyxl; hardcode totals
│   ├── compliance-reviewer.md    # check the filled form against policy.md (read-only)
│   └── deadline-watcher.md       # triage pending receipts by urgency
├── skills/
│   ├── ingest-policy/            # (setup) policy document → rules/policy.md
│   ├── update-document/          # (documents) refresh a CV/bio from its sources of truth
│   ├── intake/ · new-claim/ · file-claim/ · reconcile/ · submissions-status/ · currency-convert/
├── references/document-sources.md  # where each maintained document pulls its content (CV example inside)
├── templates/  (submission-audit · submission-folder-readme · ledger + shared core)
├── hooks/  · settings.json
```

## How to use it

1. **Copy** this folder's `.claude/` into your project: `cp -R "administrative-workflow/.claude" /path/to/project/.claude`
2. **Generate your institution's rulebook (the portable part):** drop your reimbursement policy document (PDF/Word) into `policy_source/` and run **`/ingest-policy policy_source/<doc>`**. The `policy-parser` agent reads it and fills `rules/policy.md` in the rule-ID/severity schema `compliance-reviewer` enforces — citing the source for each rule and flagging anything ambiguous for you to confirm. Moving institutions later = drop the new policy and re-run. (You can still hand-edit `rules/policy.md` instead.)
3. **Fill the remaining placeholders** — every `[YOUR …]` / `[…]` token:
   - `CLAUDE.md`: name, email, institution, admin contact, budget codes, form-cell map.
   - `agents/form-filler.md`: map your expense form's identity cells, section row-ranges, and total cells.
4. **Set up the project root:** create `pending_receipts/` and `policy_source/`, drop in the blank expense form, copy `templates/ledger.md` → `submissions.md`.
5. **For document maintenance (e.g. CV):** declare the document and its sources of truth in `references/document-sources.md` (a CV block is pre-filled as an example — point it at your `Refereeing/`, reimbursement `conference_expenses/` + `submissions.md`, webpage, etc.), then run `/update-document` to refresh it. This is the natural home for it, since the CV pulls from the reimbursement records this template already manages.
6. **Skills:** this repo's `skills/` library (installed once to `~/.claude/skills` — see the top-level [README](../../README.md)) plus the workflow-specific ones bundled here.

## The loops

- **Reimbursements:** `/intake` a receipt → `/new-claim` scaffolds the folder + form → `/file-claim` fills line items, runs `compliance-reviewer`, scores → you print/sign/hand off → `/reconcile` and `/submissions-status` keep the ledger honest.
- **Documents:** `/update-document` pulls from the sources map → reconciles against the newest version → asks you about every ambiguous item (presented-vs-attended, status changes) → writes a new dated version → rebuilds and verifies.

## Notes

- `settings.json` ships with no permission overrides, so Claude Code uses its standard prompting — it asks before editing files or running commands. Loosen it once you trust the workflow.
- This template assumes a spreadsheet (.xlsx) form. For PDF forms, swap `form-filler` for a PDF-fill approach (the `pdf` skill can help).
